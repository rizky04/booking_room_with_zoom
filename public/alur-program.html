Tentang Sistem
Sistem booking ruang meeting berbasis web yang memungkinkan karyawan melakukan reservasi ruang secara mandiri melalui form publik tanpa login. Admin dapat mengelola seluruh booking, ruang, dan membuat booking langsung. Sistem terintegrasi dengan Zoom API untuk pembuatan meeting otomatis dan email SMTP untuk notifikasi.
👤 Alur User (Publik)
  1. Buka form booking
  2. Isi data & pilih ruang/waktu
  3. Submit → status PENDING
  4. Terima email verifikasi
  5. Klik link → status CONFIRMED
  6. Terima email konfirmasi + Zoom
🔧 Alur Admin
  1. Login ke dashboard admin
  2. Pantau semua booking
  3. Buat booking langsung (confirmed)
  4. Batalkan booking + kirim email
  5. Kelola data ruang (CRUD)
  6. Lihat laporan bulanan
Modul File Utama Fungsi
BookingService app/Services/BookingService.php Buat, verifikasi, batalkan booking. Semua operasi kritis dalam DB transaction + lock.

Perlindungan dari Bentrokan
Semua skenario yang mungkin terjadi dan cara sistem mengatasinya
# Skenario Mekanisme Perlindungan Lapisan Status
1 Double booking ruang
2 orang booking ruang & waktu yang sama		 DB::transaction() + lockForUpdate() + isRoomAvailable() Database level ✅ AMAN
2 Race condition
Submit bersamaan di milidetik yang sama		 MySQL row lock memaksa antrian. Yang ke-2 menunggu, lalu cek ulang ketersediaan Database level ✅ AMAN
3 Verifikasi terlambat
Slot sudah diambil orang lain saat user verify		 isRoomAvailable(excludeId) dijalankan ulang di dalam transaction saat verify Aplikasi level ✅ AMAN
4 Token kadaluarsa
User verify setelah 24 jam		 Cek created_at + 24 jam sebelum proses verifikasi Aplikasi level ✅ AMAN
5 Zoom akun 1 bentrok
2 booking di jam yang sama butuh Zoom		 selectAccount() deteksi overlap, otomatis pakai Akun 2 Aplikasi level ✅ AMAN
6 Hapus Zoom dengan akun salah
Meeting dibuat akun 2, dihapus pakai akun 1		 account_index disimpan di zoom_meetings, selalu diteruskan ke deleteMeeting() Database level ✅ AMAN
7 Booking cancelled memblok slot
Setelah cancel, slot tetap terblok		 Semua query filter status IN ('pending','confirmed') — cancelled diabaikan Aplikasi level ✅ AMAN
